Compress Image

Technical Documentation


This application is built using FastAPI, chosen for its speed and efficiency in handling API requests. It uses Jinja2 for rendering dynamic HTML templates and serves static files like CSS and JavaScript for an interactive user experience.


Overview

URL: /upload/
Method: POST
Input: An image file (e.g., PNG, JPEG).
Output: Compressed WebP image.


Key Features

Validation: Ensures the uploaded file is an image, else returns a 400 Bad Request.
Compression: Converts the image to WebP format with high quality and near-lossless compression.

Responses:
200 OK: Returns the compressed WebP image.
400 Bad Request: If the file is not an image.
500 Internal Server Error: If processing fails.


Dependencies

Pillow: For image processing.
FastAPI: For API routing and response handling.
